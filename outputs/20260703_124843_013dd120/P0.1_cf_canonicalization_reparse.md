# P0.1 Acceptance Check — Offline Re-parse of Pilot CF Responses

**Date:** 2026-07-04
**Method:** Re-parsed all 90 stored `raw_counterfactual` responses from `instance_results.jsonl`
offline (no API calls) using the fixed `Parser.parse_counterfactual` (canonical
tokenization: clitics split, punctuation detached, casefolded, applied identically
to both sides before the Levenshtein ratio and the difflib diff). Compared the
re-parsed `cf_rules_compliant` against the value the original (pre-fix) pipeline
produced for the same raw response.

## AG News regressions are a second, correct catch — not a bug

7 AG News instances flipped True→False. Manual inspection of every one shows the
**same underlying mechanism as P0.1, in the opposite direction**: the original text
has no punctuation between the last real word and where the model appended a new
clause (an AG News quirk — headline concatenated directly to body with no
separator), so under raw-`.split()` diffing, a token like `companies.` (original)
vs `companies,` (rewritten, now followed by an appended clause) were compared as
different raw strings and the OLD code spuriously attributed the unchanged word
(`companies`) as "changed evidence" due to the punctuation mismatch alone. The
canonical diff correctly sees `companies` as unchanged and the true edit as a
**pure suffix insertion** (e.g. `ag_news_test_001223`: `... Japanese companies.` →
`... Japanese companies, the World Bank reported.` — insert `, the world bank
reported`, no original token replaced/deleted). This is exactly the existing
"insertion-only, no attribution" rule (`test_insertion_only_flip_has_no_original_attribution`)
correctly firing where it previously didn't. Net effect: AG News's canonical CF
validity rate drops because several previously-"compliant" CFs were never
minimal-edit attributions to begin with.

## One MNLI boundary case (expected, not a bug)

`mnli_validation_matched_006492` (both nova-pro and deepseek): edit is `"didn't
realize"` → `"realized"`. Canonical tokenization splits `"didn't"` into `["did",
"n't"]` (2 morphemes, matching PTB/Treebank convention — the same convention the
fix aligns SST-2 to), so the edit is now 3 canonical tokens replaced by 1 (ratio
0.333) versus 2 raw whitespace tokens replaced by 1 under the old convention (ratio
0.286) — pushing it just over the 0.3 threshold. This is a deterministic,
model-agnostic consequence of adopting one linguistically-standard tokenization
(not a reintroduction of the spacing-style confound: it fires identically
regardless of which model wrote the rewrite, unlike the original bug which
depended on whether a model happened to mimic Treebank spacing).

## Conclusion

Acceptance criterion met: SST-2 CF rules-compliance no longer differs by an order
of magnitude across models (was 0%–90%, a 90-point/undefined-ratio spread; now
40%–100%, 60 points). The remaining spread is attributable to genuine model
differences in producing minimal edits, not a tokenization artifact. No further
action needed before the 200-run; re-run this same offline check after the 200-run
completes as a regression guard if desired.
